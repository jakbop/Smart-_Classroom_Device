# Smart Classroom Device

智慧教室设备管理系统 - 基于STM32F103CBT6的物联网终端

---

## 📋 项目简介

本项目是一个智慧教室物联网终端系统，由三大部分组成：

- **STM32 终端设备**：基于 STM32F103CBT6，负责传感器采集、设备控制、OLED 显示
- **设备云服务器**：部署在阿里云 ECS，TCP 服务器 + MQTT 客户端，实现协议转换网关
- **QT 上位机**：PC 端串口控制程序，直连 STM32 进行数据展示与设备控制

## 📡 通信协议

### 上行数据格式 (STM32 → 云端/QT)

```
设备ID/类型/参数 值\n
```

| 示例 | 说明 | 上传通道 |
|------|------|----------|
| `wengyuanhang/sensor/Temp 28.22_65\n` | 上报温度_湿度 | UART1 + UART2 |
| `wengyuanhang/sensor/Light 1234\n` | 上报光照数据 | UART1 + UART2 |
| `wengyuanhang/state/Led1 1\n` | 上报 LED1 状态 (1=开, 0=关) | UART2 |
| `wengyuanhang/state/Led2 0\n` | 上报 LED2 状态 | UART2 |
| `wengyuanhang/state/Buzzer 1\n` | 上报蜂鸣器状态 (1=开, 0=关) | UART1 + UART2 |

**设备ID**: `wengyuanhang` (可在 `main.c` 中修改)

## 🔌 硬件连接

### STM32F103CBT6 引脚定义

| 功能 | 引脚 | 说明 |
|------|------|------|
| **LED1** | PB0 | LED1 控制引脚（高电平有效） |
| **LED2** | PB1 | LED2 控制引脚（低电平有效） |
| **UART1_TX** | PA9 | 发送到 QT 上位机 |
| **UART1_RX** | PA10 | 接收来自 QT 上位机 |
| **UART2_TX** | PA2 | 发送到 ESP8266 |
| **UART2_RX** | PA3 | 接收来自 ESP8266 |
| **I2C_SCL** | PB6 | OLED 时钟线 |
| **I2C_SDA** | PB7 | OLED 数据线 |
| **ADC_IN** | PA4 | 光照传感器模拟输入 (ADC1 通道4) |
| **DHT11** | PA5 | 温湿度传感器数据线 |
| **DS18B20** | PA7 | 温度传感器数据线（未启用） |
| **BUZZER** | PA8 | 无源蜂鸣器 PWM 输出 (TIM1_CH1) |

### 通信参数

| 参数 | 值 |
|------|-----|
| **UART 波特率** | 115200 |
| **数据位** | 8 |
| **停止位** | 1 |
| **校验位** | 无 |
| **TCP 端口** | 8866 |
| **I2C 速率** | 400kHz (快速模式) |
| **PWM 频率** | 1kHz (蜂鸣器) |

## 🚀 快速开始

### 开发环境

| 组件 | 工具 |
|------|------|
| **STM32** | Keil MDK-ARM V5.32 + STM32CubeMX 6.16.0 |
| **QT 上位机** | Qt Creator (Qt5) |
| **云服务器** | GCC (Linux) + Mosquitto MQTT Broker |

### 编译和烧录 (STM32)

1. 打开 `MDK-ARM/smart_cbt6.uvprojx` 工程文件
2. 点击编译按钮 (F7)
3. 连接 ST-Link 调试器
4. 点击下载按钮 (F8)

### 编译和运行 (QT 上位机)

1. 用 Qt Creator 打开 `QT_Device_Control/DeviceCtrl_UART_V1/DeviceCtrl_UART.pro`
2. 编译并运行
3. 选择串口并点击"打开"

### 编译和运行 (云服务器)

1. 将 `device_cloud-master/` 上传到阿里云 ECS
2. 编译：`gcc -o device_cloud device_server.c mqtt_client.c hash_table.c -lpthread -lmosquitto`
3. 运行：`./device_cloud`
4. 确保 Mosquitto 服务已启动

## 🔄 数据流程

### 数据上报流程

```
1. 主循环每1秒执行一次
2. DHT11 采集温湿度数据
3. ADC 采集光照值
4. 读取 LED1、LED2、蜂鸣器状态
5. 格式化为协议字符串
6. 通过 UART1 发送到 QT 上位机
7. 通过 UART2 发送到 ESP8266
8. ESP8266 通过 TCP 发送到阿里云服务器
9. TCP 服务器转换为 MQTT 发布
10. H5/小智AI 订阅并显示数据
```

### 控制命令流程

```
1. H5/小智AI 发布控制指令到 MQTT / QT 通过 UART1 发送命令
2. TCP 服务器订阅并接收 MQTT 指令
3. TCP 服务器通过 TCP 转发到 ESP8266
4. ESP8266 通过 UART2 转发到 STM32
5. STM32 触发 UART 接收中断
6. 回调函数执行 LED/蜂鸣器控制，更新 buzzer_state 标志
7. 主循环上报最新状态
```

else if(cmd == 'e'){

3. 在 `main.c` 中包含头文件并调用
4. 在主循环中添加数据采集和上传逻辑

### 添加新的状态上报

1. 在 `usart.c` 中添加状态标志变量
2. 在 `usart.h` 中声明 `extern`
3. 在 `main.c` 主循环中读取标志并上传
4. 在 QT 上位机 `recvedData()` 中添加解析逻辑

## ⚠️ 注意事项

1. ESP8266 需要预先配置好 WiFi 连接和 TCP 客户端
2. 阿里云 TCP 服务器需要同时运行 TCP 服务和 MQTT 客户端
3. MQTT 服务器地址和端口需要在 TCP 服务器中配置
4. 设备 ID 需要在整个系统中保持唯一
5. 确保所有模块共地，避免通信问题
6. 无源蜂鸣器需要 PWM 驱动，不能直接 GPIO 输出
7. DHT11 采样间隔至少 1 秒，当前主循环周期为 1 秒
8. LED1 高电平有效，LED2 低电平有效，控制时需注意
