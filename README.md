# Smart Classroom Device

智慧教室设备管理系统 - 基于STM32F103CBT6的物联网终端

---

## 📋 项目简介

本项目是一个智慧教室物联网终端设备，基于STM32F103CBT6微控制器，实现环境监测、设备控制和云端数据交互功能。

---

## 🎯 功能特性

| 功能 | 状态 | 说明 |
|------|------|------|
| **光照监测** | ✅ 已实现 | 通过ADC采集光照强度 |
| **温度监测** | ⚠️ 部分实现 | DS18B20驱动已编写，主程序暂未启用 |
| **LED控制** | ✅ 已实现 | 2路LED独立控制 |
| **OLED显示** | ✅ 已实现 | 0.96寸I2C屏幕，实时显示数据 |
| **串口通信** | ✅ 已实现 | UART2与ESP8266通信 |
| **云平台对接** | ✅ 已实现 | 数据上报和远程控制 |

## 📡 通信协议

### 上行数据格式 (STM32 → 云端)

```
设备ID/类型/参数 值\n
```

| 示例 | 说明 |
|------|------|
| `wengyuanhang/sensor/Temp 28.22\n` | 上报温度数据 |
| `wengyuanhang/sensor/Light 1234\n` | 上报光照数据 |
| `wengyuanhang/state/Led1 1\n` | 上报LED1状态 (1=开, 0=关) |
| `wengyuanhang/state/Led2 0\n` | 上报LED2状态 |

**设备ID**: `wengyuanhang` (可在 `main.c` 第 42 行修改)

---

### 下行控制命令 (云端 → STM32)

| 命令 | 功能 | LED1 | LED2 |
|------|------|------|------|
| `'a'` | 模式A | 开 | 关 |
| `'b'` | 模式B | 关 | 开 |

## 🔌 硬件连接

### STM32F103CBT6 引脚定义

| 功能 | 引脚 | 说明 |
|------|------|------|
| **LED1** | PB0 | LED1控制引脚 |
| **LED2** | PB1 | LED2控制引脚 |
| **UART2_TX** | PA2 | 发送到ESP8266 |
| **UART2_RX** | PA3 | 接收来自ESP8266 |
| **I2C_SCL** | PB6 | OLED时钟线 |
| **I2C_SDA** | PB7 | OLED数据线 |
| **ADC_IN** | 未明确 | 光照传感器输入 |
| **DS18B20** | PA7 | 温度传感器数据线 |

### 通信参数

| 参数 | 值 |
|------|-----|
| **UART波特率** | 115200 |
| **数据位** | 8 |
| **停止位** | 1 |
| **校验位** | 无 |
| **TCP端口** | 8866 |

## 🔄 数据流程

### 数据上报流程

```
1. 主循环每1秒执行一次
2. 采集光照值 (ADC转换)
3. 读取LED状态
4. 格式化为协议字符串
5. 通过UART发送到ESP8266
6. ESP8266通过TCP发送到阿里云服务器
7. TCP服务器转换为MQTT发布
8. H5页面订阅并显示数据
```

### 控制命令流程

```
1. H5页面发布控制指令到MQTT
2. TCP服务器订阅并接收指令
3. TCP服务器通过TCP转发到ESP8266
4. ESP8266通过UART转发到STM32
5. STM32触发UART接收中断
6. 回调函数执行LED控制
7. 更新状态并上报
```

---

## ⚠️ 注意事项

1. ESP8266需要预先配置好WiFi连接和TCP客户端
2. 阿里云TCP服务器需要同时运行TCP服务和MQTT客户端
3. MQTT服务器地址和端口需要在TCP服务器中配置
4. 设备ID需要在整个系统中保持唯一
5. 确保所有模块共地，避免通信问题
