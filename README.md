# Smart Classroom Device

智慧教室设备管理系统 - 基于STM32F103CBT6的物联网终端

---

## 项目简介

本项目是一个智慧教室物联网终端系统，采用三层架构：

- **STM32 终端设备**：基于 STM32F103CBT6，负责传感器采集、设备控制、OLED 显示
- **设备云服务器**：部署在阿里云 ECS，TCP 服务器 + MQTT 客户端，实现协议转换网关
- **应用层**：QT 上位机（本地串口控制）+ 小智 AI 语音助手（远程 MQTT 控制）

## 通信协议

### 自定义TCP协议格式

```
格式: "主题 载荷\n"
分隔: 空格分隔主题和载荷，'\n'为行结束符
设备ID: 主题中第一个'/'之前的部分

示例:
  "wengyuanhang/sensor/Temp 28.22_65\n"     ← 温湿度上报
  "wengyuanhang/sensor/Light 1234\n"         ← 光照上报
  "wengyuanhang/state/Led1 1\n"              ← LED1状态上报
  "wengyuanhang/state/Buzzer 0\n"            ← 蜂鸣器状态上报
```

### 上行数据格式 (STM32 → 云端/QT)

| 示例 | 说明 | 上传通道 |
|------|------|----------|
| `wengyuanhang/sensor/Temp 28.22_65\n` | 上报温度_湿度 | UART1 + UART2 |
| `wengyuanhang/sensor/Light 1234\n` | 上报光照数据 | UART1 + UART2 |
| `wengyuanhang/state/Led1 1\n` | 上报 LED1 状态 (1=开, 0=关) | UART1 + UART2 |
| `wengyuanhang/state/Led2 0\n` | 上报 LED2 状态 | UART1 + UART2 |
| `wengyuanhang/state/Buzzer 1\n` | 上报蜂鸣器状态 (1=开, 0=关) | UART1 + UART2 |

**设备ID**: `wengyuanhang` (可在 `main.c` 中修改)

### 下行控制命令 (云端/QT → STM32)

| 命令 | 功能 | LED1 | LED2 | 蜂鸣器 |
|------|------|------|------|--------|
| `'a'` | 开灯 | 开 | 关 | - |
| `'b'` | 关灯 | 关 | 开 | - |
| `'c'` | 开蜂鸣器 | - | - | 开(PWM) |
| `'d'` | 关蜂鸣器 | - | - | 关(PWM) |
| `'e'` | 开风扇 | - | - | - |
| `'f'` | 关风扇 | - | - | - |
| `'a1'` | 开智能插座 | - | - | - |
| `'b1'` | 关智能插座 | - | - | - |

> 命令 `e`/`f`/`a1`/`b1` 已在 MCP 服务器中定义，STM32 端待扩展实现。

### MQTT 主题设计

```
wengyuanhang/
├── sensor/
│   ├── Temp          ← 温湿度数据 (payload: "28.22_65")
│   ├── Light         ← 光照数据   (payload: "1234")
│   └── data          ← 插座数据   (payload: "dlsj=220.0,1.5,330.0")
├── state/
│   ├── Led1          ← LED1状态   (payload: "0" 或 "1")
│   ├── Led2          ← LED2状态   (payload: "0" 或 "1")
│   └── Buzzer        ← 蜂鸣器状态 (payload: "0" 或 "1")
└── cmd               ← 控制命令   (payload: "a"/"b"/"c"/"d"/...)
```

## 数据流程

### 传感器数据上行流程

```
1. 主循环每1秒执行一次
2. DHT11 采集温湿度数据
3. ADC 采集光照值
4. 读取 LED1、LED2、蜂鸣器状态
5. 格式化为协议字符串
6. 通过 UART1 发送到 QT 上位机
7. 通过 UART2 发送到 ESP8266
8. ESP8266 通过 TCP:8866 发送到阿里云服务器
9. 设备云网关解析协议，提取主题和载荷
10. 设备云网关通过 MQTT 发布到 Broker
11. 小智 MCP 服务器 / H5 页面订阅并显示数据
```

### 控制命令下行流程

**远程控制（小智AI / H5）：**

```
1. 用户语音指令 → 小智AI → WebSocket → MCP服务器
2. MCP服务器调用工具函数，发布 MQTT 消息到 wengyuanhang/cmd
3. Mosquitto Broker 转发消息
4. 设备云网关订阅 +/cmd 收到消息
5. 从主题提取设备ID，哈希表查找对应 TCP Socket
6. 通过 TCP:8866 转发命令到 ESP8266
7. ESP8266 通过 UART2 转发到 STM32
8. STM32 触发 UART 接收中断，执行设备控制
9. 主循环上报最新状态
```

**本地控制（QT上位机）：**

```
1. 用户点击按钮
2. QT 通过 UART1 发送单字符命令 (a/b/c/d)
3. STM32 触发 UART1 接收中断，执行设备控制
4. STM32 上报最新状态，QT 解析并更新按钮
```

### 设备云网关协议转换机制

```
                    上行：传感器数据
TCP接收 ──► 解析"主题 载荷" ──► 提取设备ID ──► 注册哈希表 ──► MQTT发布
                                                            (topic, payload原样转发)

                    下行：控制命令
MQTT订阅(+/cmd) ──► 提取设备ID ──► 哈希表查找Socket ──► TCP发送 ──► ESP8266
```

## 快速开始

### 开发环境

| 组件 | 工具 |
|------|------|
| **STM32** | Keil MDK-ARM V5.32 + STM32CubeMX 6.16.0 |
| **QT 上位机** | Qt Creator (Qt5) |
| **云服务器** | GCC (Linux) + Mosquitto MQTT Broker |
| **小智 MCP** | Python 3.x + FastMCP + Paho-MQTT |

### 编译和烧录 (STM32)

1. 打开 `MDK-ARM/smart_cbt6.uvprojx` 工程文件
2. 点击编译按钮 (F7)
3. 连接 ST-Link 调试器
4. 点击下载按钮 (F8)

### 编译和运行 (QT 上位机)

1. 用 Qt Creator 打开 `QT_Device_Control/DeviceCtrl_UART_V1/DeviceCtrl_UART.pro`
2. 编译并运行
3. 选择串口并点击"打开"

### 编译和运行 (云服务器)

1. 将 `device_cloud-master/` 上传到阿里云 ECS
2. 编译：`gcc -o device_cloud main.c device_server.c mqtt_client.c hash_table.c -lpthread -lmosquitto`
3. 确保 Mosquitto 服务已启动
4. 运行：`./device_cloud`

### 运行小智 MCP 服务器

1. 安装依赖：`pip install fastmcp paho-mqtt python-dotenv websockets`
2. 配置环境变量：`$env:MCP_ENDPOINT = "<小智云端WebSocket地址>"`
3. 启动管道：`python mcp_pipe.py my_mcp_server`
4. MCP 服务器会自动连接 MQTT Broker (39.104.71.92:1883)

## 蜂鸣器状态跟踪方案

无源蜂鸣器使用 PWM 驱动，无法通过 GPIO 读取状态，因此采用**软件标志位**方案：

- 在 `usart.c` 中定义全局变量 `uint8_t buzzer_state`
- 开启 PWM 时设置 `buzzer_state = 1`
- 关闭 PWM 时设置 `buzzer_state = 0`
- 主循环中读取 `buzzer_state` 并通过 UART 上报
- QT 上位机和小智 MCP 均可获取蜂鸣器当前状态

## 技术栈

| 技术 | 用途 |
|------|------|
| STM32 HAL 库 | 硬件抽象层 |

| Paho-MQTT | Python MQTT 客户端 |
| pthread | 云服务器多线程 |
| Mosquitto | MQTT Broker |
| 哈希表 | 设备ID→Socket映射 (最大10000设备) |

## 注意事项

1. ESP8266 需要预先配置好 WiFi 连接和 TCP 客户端
2. 阿里云 TCP 服务器需要同时运行 TCP 服务和 MQTT 客户端
3. MQTT 服务器地址和端口需要在 TCP 服务器中配置（`tcp://127.0.0.1:1883`）
4. 设备 ID 需要在整个系统中保持唯一
5. 确保所有模块共地，避免通信问题
6. 无源蜂鸣器需要 PWM 驱动，不能直接 GPIO 输出
7. DHT11 采样间隔至少 1 秒，当前主循环周期为 1 秒
8. LED1 高电平有效，LED2 低电平有效，控制时需注意
9. 小智 MCP 服务器未订阅蜂鸣器状态主题，`query_all_status()` 中不包含蜂鸣器状态
10. 命令 `e`/`f`/`a1`/`b1` 已在 MCP 服务器定义，STM32 端待扩展实现
